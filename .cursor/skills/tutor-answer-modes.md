# Mode jawaban asli vs tuned (chat WA)

Dokumen ini melengkapi [tutor-shared-style.md](./tutor-shared-style.md). Isinya aturan **lapisan gaya** supaya jawaban bisa cepat kayak chat atau sedikit dirapikan buat forum atau tugas yang dibaca dosen tanpa jadi formal berlebihan.

---

## Kapan pakai yang mana

| Mode | Cocok untuk |
| ---- | ----------- |
| **Asli** | Latihan pribadi catatan cepat jawaban pertama biar ide keluar |
| **Tuned** | Forum diskusi tuton pengumpulan tugas atau kalau kamu minta versi yang dibaca orang lain tetap santai tapi rapi |

Default bantu AI bisa mulai dari **asli** lalu tawarkan **tuned** satu blok atau sebaliknya kalau kamu bilang langsung mau yang tuned.

---

## Mode asli (WhatsApp cepat)

Tujuan kecepatan dan natural kayak ketik ke teman.

| Aturan | Penjelasan |
| ------ | ---------- |
| Tulis aja | Pakai aja bukan saja |
| Tanpa strip panjang | Narasi jangan pakai minus atau em dash di tengah kalimat ganti spasi atau pecah kalimat |
| Kurangi kutip | Jarang tanda petik ganda kecuali istilah singkat |
| Polos | Jarang bold italic code fence kecuali kode beneran |
| Koma | Boleh dikurangi kalau masih kebaca jangan sampe susah |
| Baris baru | Satu ide satu baris atau blok pendek enak dibaca di HP |

Contoh pola buruk vs oke sama seperti di tutor shared style §4 dulu: paragraf formal berat vs kalimat pendek jelas.

## Mode tuned (rapi dikit tetap ringan)

Sama struktur §2 tutor shared style salam Menurut saya isi penutup Terimakasih Sumber BMP. Bedanya **cara mengetik** supaya tidak terlalu informal tapi juga tidak kaku seperti makalah.

| Dimensi | Asli | Tuned |
| ------- | ---- | ----- |
| **Koma** | Sedikit koma biar cepat | Tambah koma di jeda napas biar enak dibaca kerasa alur |
| **Slang kasar** | Misalnya doang | Ganti ke aja atau kata netral |
| **Kata kerja** | Bentuk lisan ringan misalnya nerealisasi kepeta | Boleh lebih wajar tertulis misalnya merealisasikan tertata |
| **Sambung** | banyak dan | Kadang serta atau struktur kalimat dipotong biar jelas |
| **Istilah panjang** | Cukup di kalimat | Boleh sisip penjelasan singkat dalam kurung misalnya SDM setelah manusia kalau perlu |
| **Awal blok** | Boleh huruf kecil | Awal paragraf ide baru pakai huruf besar |
| **Baris contoh** | contoh e commerce lalu lanjut | Boleh pakai contoh e commerce : lalu jelasin setelah titik dua |
| **Kalimat penutup inti** | Sangat singkat | Tetap singkat boleh dikembangkan sedikit tanpa ganti makna kalau perlu |
| **Salah ketik formal** | terimakasih | Boleh konsisten Terimakasih, di baris sendiri sebelum Sumber |

Versi tuned **bukan** wajib mengubah isi argumen hanya merapikan alur baca dan pilihan kata.

Aturan **istilah teknis** (misalnya API Contract tetap Inggris) dan pemetaan **titik ekstensi atau integrasi** ke **integration partner** ikut [tutor-shared-style.md](./tutor-shared-style.md) bagian Bahasa dan kosakata.

## Contoh pasangan singkat dari materi yang sama

Soal tentang pendekatan sosio teknis di SIM dan keseimbangan teknologi manusia.

**Cuplikan asli**

pendekatan sosio teknikal itu SIM dilihat sebagai satu paket manusia proses data teknologi bukan cuma aplikasi atau infrastruktur doang

**Cuplikan tuned**

Pendekatan sosio teknikal itu SIM dilihat sebagai satu paket manusia proses data teknologi bukan cuma aplikasi atau infrastruktur aja

Perbedaan yang kelihatan huruf besar di awal doang ke aja.

**Cuplikan asli**

harus ada pemahaman masalah dulu terus ide solusi yang jelas baru dipilih teknologi yang cocok buat nerealisasi ide itu

**Cuplikan tuned**

harus ada pemahaman masalah dulu terus ide, solusi yang jelas baru, dipilih teknologi yang cocok buat merealisasikan ide itu

Perbedaan koma di jeda ide kata kerja lebih wajar.

## Penutup Terimakasih dan Sumber BMP

Urutan yang dipakai di tutor shared style:

Terimakasih,

Sumber : BMP [nama mata kuliah] modul [x]

Nomor modul ikut BMP kamu biasanya satu angka per modul kalau ada sub bagian tulis sesuai petunjuk dosen misalnya modul 1 bagian 3 jangan ngarang format kalau tidak ada di BMP.
